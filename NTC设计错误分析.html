⚠️ NTC 测温电路设计错误分析报告

项目：摩托车液冷系统主控板  |  NTC 型号：ZY103（R₂₅=10kΩ，B=3950，1%） |  设计目标：0~60°C

1. 结论

❌ 错误核心：运放偏置电路把有效测量窗口压缩到了 46~57°C，只有 11°C 宽。
0°C ~ 45°C 的所有温度全部被映射到同一个 ADC 值（≈700），57°C 以上同样饱和（≈2736），无法区分。

2. 电路原理与问题定位

2.1 实际电路（从网表还原）

         NTC-VCC (A3V1=3.3V)
              │
         ┌────┘
         │ NTC (ZY103, 10kΩ@25°C)
         │
    Vp ──┼──────────────────── R_pulldown (R7=8.25kΩ) ── PGND
         │
         └── U5/U8 IN+（运放正输入）


    NTC-VCC (3.3V) ── R_bias (R10 = 3kΩ) ──┬── U5/U8 IN-（运放负输入）
                                             │
                           PGND ── R_lower (R11 = 10kΩ) ──┘
                                             │
                           Vout ── R_fb   (R14 = 10kΩ) ──┘

    Vout ──► STM32 ADC（PA0~PA7）

2.2 运放输出公式推导

IN- 节点 KCL（虚短条件：VIN- = VIN+ = Vp）：

  (3.3V − Vp) / 3kΩ + (Vout − Vp) / 10kΩ + (0 − Vp) / 10kΩ = 0

化简得：
  Vout = (16 × Vp − 33) / 3

其中：  Vp = 3.3V × 8.25 / (R_NTC + 8.25)   （单位 kΩ）

ADC 值（12位）：  ADC = Vout / 3.3 × 4095

2.3 运放工作区间分析

运放从低端饱和进入线性区的条件：Vout > V_low_sat ≈ 0.56V

解方程：(16 × Vp − 33) / 3 = 0  →  Vp = 2.0625V
对应 R_NTC = 3.3×8.25/2.0625 − 8.25 = 4.33kΩ
查 RT 表：4.33kΩ 对应 ≈ 45°C

解方程：(16 × Vp − 33) / 3 = 3.3  →  Vp = 2.681V
对应 R_NTC = 3.3×8.25/2.681 − 8.25 = 2.86kΩ
查 RT 表：2.86kΩ 对应 ≈ 56°C

也就是说，运放只在 45~56°C 这一区间内处于线性放大状态，其他温度全部饱和。

4. 可视化：温度 → ADC 映射

5. 错误根因：运放偏置参数选错

5.1 偏置网络的作用

IN− 端的三个电阻决定了运放的"工作中心点"：

电阻连接值作用
R_bias (R10)NTC-VCC → IN−3kΩ设定偏置参考电压 ❌ 太小
R_lower (R11)IN− → PGND10kΩ设定偏置参考电压
R_fb (R14)Vout → IN−10kΩ负反馈，决定增益

5.2 偏置参考电压计算

运放饱和时（无反馈），IN− 的电压由 R_bias 和 R_lower 分压决定：

V_ref = VCC × (R_lower ‖ R_fb) / (R_bias + R_lower ‖ R_fb)
      = 3.3 × (10k‖10k) / (3k + 10k‖10k)
      = 3.3 × 5k / 8k
      = 2.0625V

这意味着：只有当 Vp > 2.0625V 时，运放才从低端饱和进入线性区。

而 Vp = 2.0625V 对应 R_NTC = 4.33kΩ，也就是 约 45°C。

→ 0°C ~ 45°C 整段，运放永远处于低端饱和，Vout 被钳在 ≈0.56V，ADC 恒为 ≈700。

5.3 增益过高

增益 G = R_fb/R_bias + 1 + R_fb/R_lower
       = 10/3 + 1 + 10/10 = 5.33×

5.33 倍增益把 NTC 电压的微小变化急剧放大。0°C 到 60°C 范围内 Vp 只变化了 1.86V（0.68V → 2.54V），乘以 5.33 倍后需要 9.9V 输出摆幅，而运放只有 3.3V 电源，绝大部分区间都在饱和状态。

6. 正确设计方案

方案 A：去掉运放偏置，直接分压（推荐）最简单

分压电阻 R_pulldown = 8.25kΩ 选值本身是正确的（几何平均值 √(31.77k×2.47k)≈8.85kΩ ≈ 8.25kΩ）。

只需将运放改为单位增益缓冲器（Voltage Follower），即：

  IN+ → NTC分压点 Vp
  IN− → 直接连 Vout（短接，无 R_bias、R_lower）
  Vout = Vp → 直接送 ADC

✅ ADC 从 844（0°C）到 3151（60°C），覆盖 73% 的量程，全程单调，无饱和问题。

方案 B：修正运放偏置参数，保留放大功能适合需要高精度

目标：让 Vout 在 0°C（Vp=0.68V）时为 0V，在 60°C（Vp=2.54V）时为 3.3V。

所需增益 G = 3.3 / (2.54−0.68) = 1.77×

所需偏置：K = G × 0.68 = 1.20V

联立方程：
  G = R_fb/R_bias + 1 + R_fb/R_lower = 1.77
  K = VCC × R_fb / R_bias = 1.20

取 R_fb = 10kΩ，解得：
  R_bias = 3.3×10k/1.20 = 27.5kΩ → 选 27kΩ 原来是 3kΩ，差了 9 倍！
  R_lower = 10k/(1.77−1−10/27) = 10k/0.399 = 25.1kΩ → 选 24.9kΩ
⚠️ 此方案需更换 PCB 上的 R10（3kΩ→27kΩ）和 R11（10kΩ→24.9kΩ），当前已贴片则需飞线。

7. 软件层面的临时处理（硬件不改的情况下）

❌ 无法通过软件弥补硬件的结构性缺陷。
0°C ~ 45°C 时运放处于硬饱和，ADC 固定在 ≈700，软件无法从单一数值恢复温度信息。

如果系统工作范围确实只在 46~57°C（例如压缩机高负荷区），当前设计可勉强使用，需修改代码中的温度换算参数（已在 sensor_acq.c 中实现）。

如果需要 0~60°C 完整范围，必须修改硬件。

8. 一句话总结

R_bias（R10）选了 3kΩ，应该是 27kΩ。
这一个参数的偏差导致运放的偏置参考电压从应该的 0.68V（对应 0°C）被拉高到了 2.06V（对应 45°C）， 将整个测量窗口从 0~60°C 压缩到了 46~57°C。
