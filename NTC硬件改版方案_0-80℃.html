🔧 NTC 测温电路改版方案：0~80°C（运放缓冲器模式）

NTC：ZY103（R₂₅=10kΩ，B=3950，1%） |  运放：偏置短接 → Unity-Gain Buffer  |  目标：0~80°C

1. 改版概述

将运放从"带偏置的反相放大器"改为单位增益缓冲器（Voltage Follower），Vout 直接跟随 NTC 分压电压 Vp，同时将下拉电阻从 8.25kΩ 换为 6.2kΩ（对应 0~80°C 最优值）。

✅ 改版后：ADC 有效范围 669 ~ 3412，对应 0°C ~ 80°C，覆盖率 66.9%，全程无饱和。
指标改版前（错误）改版后
运放模式带偏置放大器（G=5.33×）单位增益缓冲器（G=1×）
测量范围46~57°C（仅11°C）0~80°C（完整80°C）
ADC 有效范围700~2736（钳位）669~3412（线性）
分辨率（平均）~170 counts/°C（仅11°C）~34 counts/°C（全程）
下拉电阻8.25kΩ（为0~60°C设计）6.2kΩ（为0~80°C最优）

2. 改版原理

2.1 运放接法变化

2.2 公式变化

改版前：Vout = (16 × Vp − 33) / 3   ← 带偏置放大，仅 46~57°C 有效

改版后：Vout = Vp = 3.3 × R_pull / (R_NTC + R_pull)   ← 直接分压，0~80°C 全程有效

其中 R_pull = 6.2kΩ

2.3 为什么选 6.2kΩ？

0°C 时 R_NTC = 31.77kΩ，80°C 时 R_NTC = 1.24kΩ
最优下拉 = √(R_min × R_max) = √(31.77k × 1.24k) = 6.28kΩ ≈ 选标准值 6.2kΩ

这使 40°C（量程中点）的 ADC 值最接近量程中间值，非线性误差最小。

5. 软件更新：sensor_acq.c

改版后运放为直接缓冲，用 RT 查表替换原来的运放反推公式：

/*
 * NTC: ZY103, R25=10kΩ, B=3950
 * 电路: NTC-VCC(3.3V) -- NTC -- IN+ -- R_pull(6.2kΩ) -- PGND
 *       运放: Unity-Gain Buffer (Vout = Vp)
 * 范围: 0~80°C, ADC 669~3412
 */
#define NTC_T_MIN     0
#define NTC_T_MAX     80
#define NTC_ADC_LOW   630     /* 略低于 0°C 的理论值 669，留余量 */
#define NTC_ADC_HIGH  3450    /* 略高于 80°C 的理论值 3412，留余量 */

/* ADC 查表：每 1°C 一个条目，索引 0=0°C … 80=80°C（共81项，ADC 从低到高） */
static const uint16_t s_ntc_table[] = {
     669,  697,  725,  755,  785,  816,  847,  880,  913,  947,  /* 0~9°C  */
     981, 1016, 1052, 1089, 1126, 1164, 1202, 1241, 1280, 1320,  /* 10~19°C */
    1361, 1401, 1442, 1484, 1526, 1568, 1610, 1652, 1695, 1738,  /* 20~29°C */
    1780, 1823, 1866, 1909, 1951, 1994, 2036, 2078, 2120, 2162,  /* 30~39°C */
    2203, 2244, 2285, 2325, 2365, 2404, 2443, 2481, 2519, 2557,  /* 40~49°C */
    2593, 2630, 2665, 2700, 2735, 2769, 2802, 2835, 2867, 2898,  /* 50~59°C */
    2928, 2959, 2988, 3017, 3045, 3072, 3099, 3125, 3151, 3176,  /* 60~69°C */
    3200, 3224, 3247, 3270, 3292, 3313, 3334, 3354, 3374, 3393,  /* 70~79°C */
    3412                                                           /* 80°C   */
};

float SensorAcq_NTCToCelsius(uint16_t raw)
{
    if (raw <= NTC_ADC_LOW)  return NAN;  /* 断路或超量程冷 */
    if (raw >= NTC_ADC_HIGH) return NAN;  /* 短路或超量程热 */

    /* 二分查找：找最大的 i 使 s_ntc_table[i] <= raw */
    int lo = 0, hi = NTC_T_MAX - NTC_T_MIN - 1;
    while (lo < hi) {
        int mid = (lo + hi + 1) / 2;
        if (s_ntc_table[mid] <= raw) lo = mid;
        else hi = mid - 1;
    }

    /* 线性插值 */
    float adc0 = s_ntc_table[lo];
    float adc1 = s_ntc_table[lo + 1];
    return (NTC_T_MIN + lo) + (raw - adc0) / (adc1 - adc0);
}
✅ 仅修改 sensor_acq.c 中的查表数据和边界常量，函数接口不变，无需修改调用方代码。

6. 备选方案：保留运放增益，充分利用 ADC 量程

若希望 0~80°C 完整对应 ADC 0~4095（充分利用 12 位精度），可保留放大功能，但修正偏置参数：

目标：Vout = 0V 在 0°C（Vp=0.539V），Vout = 3.3V 在 80°C（Vp=2.749V）
所需增益 G = 3.3 / (2.749 − 0.539) = 1.493×
所需偏置 K = G × 0.539 = 0.805V

取 R_fb = 10kΩ：
R_bias = VCC × R_fb / K = 3.3 × 10k / 0.805 = 41.0kΩ → 选 39kΩ 或 43.2kΩ
R_lower = R_fb / (G − 1 − R_fb/R_bias) = 10k / (1.493 − 1 − 0.244) = 40.2kΩ → 选 39kΩ 或 40.2kΩ
位号当前值备选方案值操作
R10/R12/... (R_bias)3kΩ39kΩ换值
R11/R13/... (R_lower)10kΩ39kΩ换值
R14/R15/... (R_fb)10kΩ10kΩ（不变）保留
R7/R8/... (R_pull)8.25kΩ6.2kΩ换值

备选方案效果：ADC ≈ 24（0°C） ~ 4080（80°C），精度约 51 counts/°C（比方案A的34提升50%）。

⚠️ 备选方案计算值（39kΩ/39kΩ）需用实测数据校准；缓冲器方案（主推）参数更确定，直接用查表即可。

1
